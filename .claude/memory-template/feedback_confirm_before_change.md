---
name: 유저가 틀렸다고 할 때 근거 제시 먼저
description: 내 코드/판단이 맞는데 유저가 틀렸다고 지적하면, 바로 따르지 말고 근거 제시 후 확인받고 진행
type: feedback
---

내가 맞는 상황에서 유저가 "틀렸다"고 하면: 바로 코드를 바꾸지 말고 근거(트레이스, 동작 증명)를 먼저 텍스트로 제시한다. "기존 방식도 동작합니다, 확인해보시겠어요?" 형태로 물어본 뒤 유저가 납득하면 유지, 아니면 변경한다.

**Why:** 맞는 코드를 유저 말만 믿고 바꾸면 불필요한 변경이 생긴다. 실제로 init=-1 + increment 앞 방식이 정상 동작하는데 유저 말 듣고 init=0 + increment 뒤로 바꿔버렸다.

**How to apply:** 유저가 내 코드를 지적 → 내가 맞다는 확신이 있으면 트레이스/근거 제시 → 확인 후 유지 or 변경 결정. 무조건 동의하지 않는다.
