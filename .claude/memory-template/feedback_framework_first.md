---
name: 프레임워크 의도 먼저 확인
description: 프레임워크 파일 수정 전 원래 의도를 파악하고, 프로젝트 편차는 이유와 함께 기록
type: feedback
---

프레임워크 문서(`.claude/docs/hooks-reference/`, `.claude/rules/`, `.claude/skills/` 등)를 수정하기 전에 반드시 원래 의도를 파악한다.

**Why:** hook 명세 문서를 확인하지 않고 내용을 지우고 리다이렉트 쪽지로 만들었다. 올바른 처리는 실제 구현 코드를 반영한 정식 명세 업데이트였다.

**How to apply:**
1. 프레임워크 파일을 건드리기 전에 같은 디렉토리의 다른 파일을 읽어 패턴을 확인한다
2. 프로젝트가 프레임워크 원래 의도에서 벗어날 때는 이유를 문서에 명시한다
3. "삭제"보다 "업데이트"를 먼저 검토한다. 삭제는 중복이 확실할 때만
