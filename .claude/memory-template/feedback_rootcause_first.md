description: 버그 수정 시 증상 억제보다 근본 원인 제거를 우선해야 한다는 피드백
type: feedback
---

버그 수정 전 반드시 물어야 할 것: "이게 증상을 막는 건가, 원인을 제거하는 건가?"

**Why:** 세이브/로드 버그 5건이 모두 같은 근본 원인(저장 타이밍 불일치)에서 파생됐는데, 각각 보정 코드로 개별 패치했다. 나중에야 근본 원인을 발견하고 타이밍 이동으로 일괄 해결.

**How to apply:**
1. 버그를 발견하면 먼저 "같은 패턴이 다른 파일에도 있나?" grep 전수 검색
2. 수정 코드가 버그를 직접 일으키는 시스템이 아닌 곳에 있다면 증상 억제 신호
3. 증상 억제 패치가 2건 이상 같은 방향이면 → 근본 원인 탐색 후 구조 변경 제안
4. coding-standards.md Code Review Checklist "버그 수정 검증" 섹션 참조
