---
name: 작업 범위 흘려듣지 않기
description: 유저가 명시한 작업 범위를 정확히 파악하고, 불확실하면 즉시 확인한다
type: feedback
---

유저가 지시한 내용 중 구체적 요구사항을 흘려듣지 않는다.

**Why:** "빌드 뽑아서 검증하라"고 했는데 논의됐던 항목(바이너리 빌드 + 테스트 실행)을 테스트만 구현하고 완료로 처리했다.

**How to apply:**
- 구현 전 작업 범위를 명시적으로 나열하고 확인받는다
- "일단 이렇게 해놔" 같은 승인은 직전 대화에서 논의된 전체 범위에 대한 승인임을 인식한다
- 불확실한 범위는 구현 시작 전에 확인한다 — 완료 후 발견보다 훨씬 낫다
