name: TD 아키텍처 감사 선행
description: 새 시스템 추가 또는 기존 시스템 간 소유권 변경 시 TD 감사 후 작업 가이드라인 하에 구현
type: feedback
---

새 시스템을 추가하거나 기존 시스템 간 소유권/역할을 변경하는 구현을 시작하기 전에 반드시 TD(technical-director) 에이전트에게 아키텍처 감사를 요청하고, 그 결과로 나온 가이드라인에 따라 구현한다.

**Why:** 시스템을 TD 감사 없이 독자 설계로 구현했다가, 상태 소유권이 여러 시스템에 분산되는 위반이 발생. 같은 데이터가 서로 다른 값을 표시하는 치명적 버그로 이어졌다.

**How to apply:**
1. 구현 요청 수신 → TD 에이전트 호출하여 아키텍처 감사
2. TD가 소유권, 인터페이스, ADR 필요 여부 결정
3. GDD 업데이트 (GDD-first 규칙과 함께 적용)
4. ADR 작성 (필요 시)
5. TD 가이드라인에 따라 코드 구현

**적용 기준 (이 중 하나라도 해당하면 TD 감사 필수):**
- 새 autoload/시스템 추가
- 기존 시스템 간 데이터 소유권 이전
- 두 시스템이 같은 종류의 데이터를 각자 생성/관리하는 구조
- 인터페이스(시그널, public API) 변경이 2개 이상 파일에 영향
