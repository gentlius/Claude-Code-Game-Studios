description: 테스트 실패 시 절대 프로덕션 코드를 테스트에 맞추지 말 것. GDD → 코드 → 테스트 순서.
type: feedback
---

테스트가 실패할 때 "초록 만들기" 위해 프로덕션 코드를 수정하는 것은 금지다.

**Why:** 통합 테스트가 stale 상태(GDD·코드가 바뀌었는데 테스트는 안 바뀐 채 방치됨)였는데, 테스트 실패를 보고 프로덕션 코드를 수정해버렸다. GDD·코드가 맞는데 테스트가 틀린 상황이었으므로 완전히 반대 방향으로 수정한 것.

**How to apply:**
1. 테스트 실패 → 즉시 GDD 확인
2. 코드가 GDD와 일치하면 → **테스트를 수정**
3. 코드가 GDD와 불일치하면 → **코드를 수정** (이 경우만 프로덕션 코드 변경)
4. 판단이 애매하면 수정 전에 유저에게 확인 요청

GDD → 코드 → 테스트 순서가 진리다. 테스트는 스펙의 증거이지 스펙 자체가 아니다.

코드·GDD 변경 시 테스트는 **같은 커밋**에서 갱신해야 한다. 미루면 다음 세션에서 방향 판단 오류 발생.

커밋 타이밍: 태스크 하나 완료 시, 테스트 그린 시, 다른 태스크로 전환 직전, 세션 마무리 시점에 커밋을 제안한다. 스프린트 전체를 한 커밋에 몰아넣는 건 금지.
