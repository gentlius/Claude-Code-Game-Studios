# 포트폴리오 UI (Portfolio UI)

> **Status**: Approved
> **Author**: user + game-designer
> **Last Updated**: 2026-03-26
> **Implements Pillar**: 체감있는 성장 (Feel the Growth), 읽는 재미 (Read the Market)

## Overview

포트폴리오 UI는 플레이어의 보유 종목, 손익, 자산 현황을 표시하는 Presentation
시스템이다. 트레이딩 스크린 하단 탭(요약 뷰)과 전체 화면 확장(상세 뷰) 두 가지
모드를 제공한다. 포트폴리오 관리 시스템의 데이터를 읽어 매 틱 실시간으로 수익률과
평가 금액을 갱신한다.

필라 "체감있는 성장"에 따라 수익률 변화가 실시간으로 눈에 보여야 한다. 빨갛게
빛나는 +15%의 쾌감, 파랗게 물드는 -8%의 긴장감 — 숫자 자체가 게임플레이의
피드백이다. 필라 "읽는 재미"에 따라 포트폴리오 정보는 한눈에 파악 가능해야 한다.

## Player Fantasy

포트폴리오 탭을 연다. 총 자산 ₩1,015,000 — 빨간 글씨로 +1.5%. 스타칩 10주,
+10.0% 빛나는 빨강. 메디진(MDG) 3주, -4.2% 차분한 파랑. 각 종목의 수익률이 내 판단의
성적표다. 시즌 시작 100만원에서 여기까지 올라왔다 — 성장이 숫자로 보인다.

거래 내역을 스크롤한다. "틱 85에서 스타칩 10주 매수 @ 65,000원" — 그때 뉴스
보고 바로 산 거였지. 그 판단이 지금 10% 수익으로 돌아왔다.

## Detailed Design

### Core Rules

#### 규칙 1. 요약 뷰 (트레이딩 스크린 하단 탭)

트레이딩 스크린의 하단 패널에서 "포트폴리오" 탭으로 표시되는 간략 뷰.

```
┌─────────────────────────────────────────┐
│ 포트폴리오 요약          3/3 종목 보유   │
├─────────────────────────────────────────┤
│ 총 평가금액: ₩1,015,000  (+1.5%)        │
│ 예수금: ₩300,000  |  주식: ₩715,000  |  시드: ₩1,000,000 │
├──────┬───────┬───────┬─────┬───────────┤
│ 종목  │ 수량  │ 현재가  │ 수익률│ 평가금액  │ S/T  │
├──────┼───────┼───────┼─────┼───────────┼──────┤
│ STC  │ 10주  │71,500 │+10.0%│  715,000  │ [설정] │
└──────┴───────┴───────┴─────┴───────────┴──────┘
```

> **참고**: 와이어프레임은 단일 종목(STC 10주) 보유 예시. 총 평가금액 ₩1,015,000 =
> 예수금 ₩300,000 + STC 평가 ₩715,000. 시드는 `SeasonManager.get_season_start_deposit()`에서 가져온다. 복수 종목 예시는 상세 뷰 참조.

- 보유 종목 리스트: 종목코드, 수량, 현재가, 수익률(%), 평가금액, **S/T 버튼**
- S/T(손절/익절) 버튼: TR2 해금 시 활성화. 클릭 시 `StopTakeSystem.set_condition()` 다이얼로그 호출
- 수익률 색상: 빨강(+), 파랑(-), 회색(0)
- 보유 종목 수 / 최대 보유 가능 표시 (예: "3/3 종목 보유")
- **예약금 표시**: 미체결 지정가 매수 예약금 합계. `reserved_cash > 0`일 때만 표시. 요약 뷰에서는 "예약: ₩200,000" 형태, 상세 뷰에서는 비중(%) 포함. 데이터: `PortfolioSummary.reserved_cash`. `reserved_cash = 0`일 때 예약 표시 영역이 사라지고(collapse), 현금 표시가 남은 공간을 채운다. 레이아웃 적응: `현금: ₩1,000,000 | 주식: ₩715,000` (예약 없이 2열).
- 종목 클릭 → 차트 전환
- "상세 보기" 버튼 → 상세 뷰 확장

#### 규칙 2. 상세 뷰 (전체 화면 확장)

요약 뷰에서 "상세 보기" 클릭 시 전체 화면으로 확장.

```
┌─────────────────────────────────────────────────┐
│  포트폴리오 상세                     [요약으로]   │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─ 자산 현황 ──────────────────────────────┐   │
│  │ 총 평가금액: ₩1,015,000  (+1.5%)           │   │
│  │ 예수금: ₩300,000 (29.6%)                  │   │
│  │ 미체결예약: ₩0 (0.0%)                     │   │
│  │ 주식: ₩715,000 (70.4%)                   │   │
│  │ 시즌 수익률: +1.5%  |  시즌 순위: 3위/10  │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  ┌─ 보유 종목 ──────────────────────────────┐   │
│  │ 스타칩 (STC) — 반도체/IT                  │   │
│  │ 보유: 10주  |  평균매수: ₩65,000          │   │
│  │ 현재가: ₩71,500  |  수익률: +10.0%        │   │
│  │ 평가금액: ₩715,000  |  손익: +₩65,000     │   │
│  │ 비중: 70.4%  |  ████████░░ 포트폴리오 비중 │   │
│  │ [차트 보기] [매도]                        │   │
│  ├──────────────────────────────────────────┤   │
│  │ ... (다른 보유 종목)                       │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  ┌─ 거래 내역 ──────────────────────────────┐   │
│  │ 최근 거래                     [전체 보기]  │   │
│  │ 틱 152 | 매수 | STC 10주 @ ₩65,000      │   │
│  │ 틱 120 | 매도 | GRC  5주 @ ₩325,000     │   │
│  │   → 실현 손익: +₩25,000                  │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

##### 2-1. 자산 현황 섹션

| 표시 항목 | 설명 | 갱신 주기 |
|----------|------|---------|
| 총 평가금액 | account_total_value (= PortfolioManager.get_total_assets()) | 매 틱 |
| 수익률 (%) | return_rate | 매 틱 |
| 예수금 + 비중 | sim_cash + cash_weight | 매매 시 |
| 미체결예약 + 비중 | reserved_cash (OrderEngine → PortfolioManager 캐시) | 주문 제출·체결·만료 시 |
| 주식 평가액 + 비중 | stock_value + stock_weight | 매 틱 |
| 시드 자본 | SeasonManager.get_season_start_deposit() | 시즌 시작 시 1회 |
| 시즌 순위 | 시즌 관리 시스템에서 제공 | 틱 단위 또는 일 단위 |

##### 2-2. 보유 종목 카드

| 표시 항목 | 설명 |
|----------|------|
| 종목명 + 섹터 | "스타칩 (STC) — 반도체/IT" |
| 보유 수량 | 정수 |
| 평균 매수가 | avg_buy_price (원) |
| 현재가 | 실시간 갱신 |
| 수익률 (%) | unrealized_pnl_pct |
| 평가금액 | quantity × current_price |
| 미실현 손익 | unrealized_pnl (원) |
| 포트폴리오 비중 | weight_i (%) + 프로그레스 바 |
| 액션 버튼 | [차트 보기] → 차트 전환, [매도] → 주문 패널 매도 모드 |

##### 2-3. 거래 내역

최근 거래를 시간순(최신 상단)으로 표시.

| 표시 항목 | 예시 |
|----------|------|
| 틱/거래일 | "틱 152 (2주차 화요일)" |
| 거래 유형 | 매수 / 매도 |
| 종목 + 수량 + 체결가 | "STC 10주 @ ₩65,000" |
| 실현 손익 (매도 시) | "+₩25,000 (+4.2%)" |

기본 5건 표시. "전체 보기" 클릭 시 스크롤 가능한 전체 내역.

#### 규칙 3. 색상 체계

| 상태 | 색상 | 사용처 |
|------|------|--------|
| 수익 (+) | 빨강 (#E74C3C) | 수익률, 손익 금액, 캔들 상승 |
| 손실 (-) | 파랑 (#3498DB) | 수익률, 손익 금액, 캔들 하락 |

| 포트폴리오 비중 바 | 진한 초록 | 비중 프로그레스 바 |

한국 주식시장 관례 (상승=빨강, 하락=파랑)를 일관 적용.

#### 규칙 4. 금액 표시 형식

| 금액 범위 | 표시 형식 | 예시 |
|----------|---------|------|
| 1,000 미만 | 원 단위 | ₩850 |

부호: 수익은 "+", 손실은 "-" 명시. 0은 부호 없음.

### States and Transitions

| State | Description | Transition |
|-------|-------------|------------|
| **SUMMARY** | 트레이딩 스크린 하단 탭. 요약 뷰 | → DETAIL (상세 보기 클릭) |
| **DETAIL** | 전체 화면 확장. 상세 뷰 | → SUMMARY (요약으로 클릭 / Esc / [매도] 클릭 시 SUMMARY로 전환 + 주문 패널 매도 모드 활성화) |
| **SETTLEMENT** | 장 마감. 일일 수익률 확정. 변동 없음. 진입: `on_market_state_changed(MARKET_CLOSED)` | → SUMMARY (다음 거래일 시작: `on_market_state_changed(PRE_MARKET)`) |

### Interactions with Other Systems

| System | Direction | Interface |
|--------|-----------|-----------|
| **포트폴리오 관리** | 포트폴리오 UI가 의존 | `get_all_holdings()` — 보유 종목 리스트. `get_total_assets()` — 총 자산. `get_return_rate()` — 수익률. `get_transaction_history(limit)` — 거래 내역 |
| **재화 시스템** | 포트폴리오 UI가 참조 | `get_sim_cash()` — 현금 잔액 |
| **가격 엔진** | 포트폴리오 UI가 참조 | `get_current_price(stock_id)` — 실시간 현재가 |
| **종목 DB** | 포트폴리오 UI가 참조 | `get_stock(stock_id)` — 종목명, 섹터 표시 |
| **트레이딩 스크린** | 트레이딩 스크린이 호스팅 | 하단 패널 탭 배치. 종목 클릭 → 차트 전환 연결 |
| **트레이딩 스크린 (주문 패널)** | 포트폴리오 UI → 트레이딩 스크린 | `[매도]` 클릭 시 `on_sell_requested(stock_id: String, quantity: int)` 시그널 발신 → 트레이딩 스크린이 수신하여 주문 패널을 매도 모드로 전환 + 종목/수량 프리필. `quantity`는 현재 전체 보유 수량 (`holding.quantity`). 플레이어는 주문 패널에서 수량 조정 가능. 상세 뷰에서 클릭 시 SUMMARY로 전환 후 발신 |
| **시즌/대회 관리** | 포트폴리오 UI가 참조 | 시즌 순위 표시 |
| **스킬 트리** | 포트폴리오 UI가 참조 | `get_max_holdings()` — 최대 보유 종목 수 표시 ("3/3 종목 보유"). **Soft** |

## Formulas

### F1. 표시용 수익률

```
display_return_pct = round(unrealized_pnl / total_invested × 100, 1)
```

| Variable | Type | Source | Description |
|----------|------|--------|-------------|
| `unrealized_pnl` | int | portfolio-manager F2 (HoldingEntry.unrealized_pnl) | 미실현 손익 금액 |
| `total_invested` | int | portfolio-manager (HoldingEntry.total_invested = avg_buy_price × quantity) | 누적 투자 금액 |

소수점 1자리 반올림. 표시: "+10.0%" / "-4.4%"

### F2. 포트폴리오 비중

```
weight_pct = round(holding_value / account_total_value × 100, 1)
cash_weight_pct = round(sim_cash / account_total_value × 100, 1)
reserved_weight_pct = round(reserved_cash / account_total_value × 100, 1)
```

`account_total_value = sim_cash + reserved_cash + Σ(holding_value)`.
전 비중의 합 (주식 + 현금 + 예약) = 100% (반올림 오차 ±0.1% 허용).

### F3. 금액 포맷팅

```
format_won(amount):
    if amount >= 0: return "₩" + format_comma(amount)
    else: return "-₩" + format_comma(abs(amount))

format_pnl(amount, pct):
    if amount > 0:
        return "+₩" + format_comma(amount) + " (+" + pct + "%)"
    elif amount < 0:
        return "-₩" + format_comma(abs(amount)) + " (-" + abs(pct) + "%)"
    else:
        return "₩0 (0.0%)"
```

### F4. 색상 결정

```
get_color(value):
    if value > 0: return RED (#E74C3C)
    if value < 0: return BLUE (#3498DB)
    return GRAY (#95A5A6)
```

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 보유 종목 0개 | 빈 상태 메시지 표시. 총 자산 = sim_cash + reserved_cash. 수익률 0% | 빈 화면 방지 |
| 전량 매도 직후 | 해당 종목 카드 즉시 제거. holding_count 갱신 | 실시간 반영 |
| 수익률 ±100% 이상 | 정상 표시. 표시 범위 제한 없음 | 극단 성공/실패 표시 |
| 시즌 첫날 (매매 없음) | 모든 비중 현금 100%. 수익률 0%. 빈 거래 내역 | 정상 초기 상태 |
| 가격 급변으로 색상 실시간 전환 | 틱마다 색상 재계산. 빨강↔파랑 전환 시 부드러운 트랜지션 없음 (즉시 전환) | 정보 정확성 > 시각 부드러움 |
| 비중 합계 반올림 오차 | ±0.1% 오차 허용. UI에서 보정하지 않음 | 단순성 |
| 거래 내역 200건 이상 | 스크롤 + 페이지네이션 (20건 단위) | 성능/가독성 |
| 시즌 종료 시 | 최종 수익률/순위 확정 표시. 거래 내역 보존 (시즌 결과 화면에서 참조) | 시즌 회고 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 포트폴리오 관리 | 포트폴리오 UI가 의존 | 보유 종목, 손익, 거래 내역 데이터. **Hard** |
| 재화 시스템 | 포트폴리오 UI가 참조 | 현금 잔액 (`sim_cash`). **Hard** |
| 주문 엔진 | 포트폴리오 UI가 간접 참조 | 예약금 (`reserved_cash`)은 `OrderEngine.get_total_reserved_cash()` → `PortfolioManager` 캐시 경유. **Hard** (예약금 표시 및 F2 총 자산 계산에 필수) |
| 가격 엔진 | 포트폴리오 UI가 참조 | 실시간 현재가. **Hard** |
| 종목 DB | 포트폴리오 UI가 참조 | 종목 표시 정보. **Soft** |
| 트레이딩 스크린 | 트레이딩 스크린이 호스팅 | 하단 탭 배치. **Hard** |
| 트레이딩 스크린 (주문 패널) | UI 내부 시그널 (시스템 레벨 아님) | `on_sell_requested(stock_id, quantity)` — 포트폴리오 UI가 발신, 트레이딩 스크린이 수신하여 주문 패널 매도 모드 전환. **Soft** |
| 시즌/대회 관리 | 포트폴리오 UI가 참조 | 시즌 순위. **Soft** (미구현 시 순위 미표시) |
| 게임 시계 | 포트폴리오 UI가 의존 | `on_tick` 시그널로 매 틱 실시간 갱신 트리거. **Hard** |

> **참고**: Player Fantasy에서 메디진(MDG) 수익률 -4.2%는 서사적 예시이며, 구현 시 실제 값은
> portfolio-manager의 F2 수식으로 계산된다.
