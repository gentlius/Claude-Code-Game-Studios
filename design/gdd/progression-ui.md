# 프로그레션 UI (Progression UI)

> **Status**: In Review
> **Author**: user + game-designer + ux-designer
> **Last Updated**: 2026-04-02
> **Implements Pillar**: 체감있는 성장 (Feel the Growth)

## Overview

프로그레션 UI는 플레이어의 레벨, 경험치, 스킬 트리 정보를 표시하고
스킬 해금 인터페이스를 제공하는 Presentation 시스템이다. 재무 데이터(자산/현금)와
성장 데이터(레벨/XP)를 시각적으로 분리하여, 트레이딩 중에는 성장 정보가
배경에 머물다가 장 마감·레벨업·시즌 종료 시점에 전면으로 부각되는
"이벤트 기반 프로미넌스" 패턴을 따른다.

## Player Fantasy

장이 끝났다. 정산 화면에 "수익률 +3.2% → 배율 ×2.0 → +60 XP"가 빛난다.
상태바 아래 XP 바가 부드럽게 차오른다. 이번 시즌 3번째 레벨업 — 상단에서
배너가 내려온다: "Lv.7 달성 — 스킬 포인트 +1". K키를 눌러 스킬 트리를 연다.
네 갈래 길에서 "보조지표"를 해금한다. 다음 장에는 RSI와 MACD가 차트에 뜬다.

필라 "체감있는 성장" — XP와 레벨업이 트레이딩 흐름을 방해하지 않으면서도,
정산 시점에 "내 판단이 성장으로 이어졌다"는 보상감을 명확히 전달한다.

## Detailed Design

### Core Rules

#### 규칙 1. XP 바 — 상시 표시 (상태바 하단)

상태바 아래에 14~16px 높이의 얇은 XP 프로그레스 바를 전체 너비로 배치한다.

```
┌─────────────────────────────────────────────────────────────┐
│ 시즌 3 | 2주차 화요일 | ■■■□ 틱 152/390 | ▶ 1x [⏸]         │
│ 총 자산: ₩1,015,000 (+1.5%) | 시드: ₩300,000                │
├──────────────────────────────────────────────────────────────┤
│ Lv 7 ▐████████████░░░░░░░░░░░░▌ 1,240 / 2,080 XP  [SP: 2] │
└──────────────────────────────────────────────────────────────┘
```

구성 요소:
- **레벨 배지**: `Lv N` 텍스트 (좌측)
- **프로그레스 바**: 현재 XP / 다음 레벨 XP 비율 (채움)
- **XP 수치**: `현재 / 필요` 텍스트 (바 우측)
- **스킬 포인트 배지**: `[SP: N]` — 미사용 포인트가 1+ 일 때만 표시. 첫 등장 시 페이드인 애니메이션, 이후 정적 표시
- **스킬 트리 버튼**: SP 배지 클릭 또는 `K`키로 스킬 트리 오버레이 열기

정보 위계 원칙: 재무 데이터(상태바)가 항상 1순위, 성장 데이터(XP 바)는 2순위.

#### 규칙 2. 일일 XP 획득 피드백 — 정산 팝업 통합

장 마감(`MARKET_CLOSED`) 시 일일 정산 팝업에 XP 행을 추가한다.

```
┌─────────────────────────────────────────────────────┐
│  2주차 화요일 정산                                    │
│                                                      │
│  총 자산:  ₩1,052,000  (+3.7%)                       │
│  일일 수익률: +3.2%                                   │
│                                                      │
│  ───────────────────────────────────────────────────  │
│  오늘의 XP                                           │
│  수익률 +3.2%                                        │
│  ┌─────┬─────┬──────┬──────┬──────┐                  │
│  │< 0% │0~1% │1~3%  │3~5% ▼│ 5%+ │  ← 구간 바       │
│  └─────┴─────┴──────┴──────┴──────┘                  │
│  기본 30 XP  ×  2.0  =  +60 XP                      │
│                                                      │
│  ▐██████████████████████████████░░░░░░░░░░░▌         │
│  1,180 → 1,240 / 2,080 XP                           │
│  ───────────────────────────────────────────────────  │
│                           [ 확인 Enter ]              │
└─────────────────────────────────────────────────────┘
```

- XP 행은 금색 하이라이트로 강조 (보상 시각 언어)
- **수익률 구간 바**: 5개 구간(`< 0%` / `0~1%` / `1~3%` / `3~5%` / `5%+`)을 가로 바로 표시.
  플레이어의 오늘 수익률 구간을 ▼ 마커 + accent 색상으로 하이라이트.
  구간 바는 XP 배율과 직접 대응하여 "왜 이 배율을 받았는지" 시각적으로 전달
- 공식 표시: `기본 N × 배율 M = +결과` — "높은 수익률 = 더 많은 XP" 학습 유도
- 팝업 내 미니 XP 바 애니메이션 (1.5초, 이전값 → 새 값)
- 팝업 닫은 후 상태바 하단 XP 바도 동기화 애니메이션
- 거래 0건인 날: XP 행 생략 또는 "거래 없음 — XP 미부여" 표시

#### 규칙 3. 레벨업 연출 — 상단 배너

레벨업 발생 시 (정산 팝업 닫은 후) 상단에서 전체 너비 배너(~80px)가 슬라이드 인:

```
┌─────────────────────────────────────────────────────────────┐
│  ★  LEVEL UP    Lv 6 → Lv 7    스킬 포인트 +1 획득          │
│  [스킬 트리 열기  K]                           [닫기  Esc]   │
└─────────────────────────────────────────────────────────────┘
```

- 배경 약간 어둡게 딤 처리 (0.3 알파)
- 블룸버그 알림 톤 — 전문적이되 눈에 띄게
- `[스킬 트리 열기]` 버튼 → 스킬 트리 풀스크린 오버레이로 직행
- `Esc` 또는 배너 외 클릭으로 닫기
- 자동 닫기 없음 — 플레이어가 명시적으로 닫아야 함
- 한 번에 2+ 레벨업 시: "Lv 5 → Lv 7 (+2) 스킬 포인트 +2 획득"

#### 규칙 4. 스킬 트리 오버레이 — 풀스크린, 자동 일시정지

> **씬 소유권**: `GrowthScreen`은 `MainScreen`의 F3(성장) 탭 자식 씬으로 인스턴스된다.
> F3 탭 구조 및 탭 전환 로직은 `league-ui.md §3-1`이 정본이다.

`K`키 또는 XP 바의 SP 배지 클릭으로 스킬 트리 풀스크린 오버레이를 연다.

> **K키 동작**: K키는 스킬 트리 오버레이를 직접 여는 단축키다 (F3 탭으로의 화면 전환이 아닌,
> 현재 탭 위에 오버레이를 띄우는 방식). 성장 화면(GrowthScreen) 자체를 열려면 F3 탭을 클릭한다.

```
┌─────────────────────────────────────────────────────────────┐
│  스킬 트리                    Lv 7 | SP: 2        [닫기 Esc] │
│  ─────────────────────────────────────────────────────────── │
│                                                              │
│  [분석 도구]    [시장 감지]    [거래 스킬]    [포트폴리오]     │
│                                                              │
│  A0 캔들차트    S0 뉴스(30s)  TR0 시장가     P0 3종목        │
│    ✓             ✓             ✓              ✓             │
│    │             │             │              │             │
│  A1 이동평균    S1 빠른뉴스   TR1 지정가     P1 5종목        │
│    ■             □             □              □             │
│    │             │             │              │             │
│  A2 보조지표    S2 실시간     TR2 손절/익절   P2 10종목       │
│    ○             ○          ┌──○              ○             │
│                  │          │  │                            │
│                S3 루머채널  TR3 공매도                        │
│                  ○             ○                            │
│                                │                            │
│                             TR4 레버리지                     │
│                                ○                            │
│                                                              │
│  * A0/S0/TR0/P0는 T0 기본 스킬로 항상 해금 상태.             │
│    skill-tree.md 규칙 1 참조.                                │
│  * A3/A4(분석), P3(포트폴리오)는 향후 확장 예정.              │
│    현재 스킬 트리: A0-A2, S0-S3, TR0-TR4, P0-P2 (14개).     │
│                                                              │
│  ───────────────────────────────────────────────────────── │
│  선택: A2 보조지표 — RSI(14), MACD(12,26,9) 하단 패널 표시   │
│  선행 조건: A1 이동평균선 ✓                                   │
│  [해금하기 — 1 SP]                                           │
└─────────────────────────────────────────────────────────────┘
```

오버레이 동작:
- 열 때 게임 자동 일시정지 (`GameClock.toggle_pause()`)
- 닫을 때 자동 일시정지 해제
- 배경: 트레이딩 화면 블러 + 딤 (게임 상태는 보이되 상호작용 불가)
- `Esc` 또는 `K`로 닫기
- 스킬 트리 오버레이가 열리면 PAUSED 상태로 전환되며, 트레이딩 스크린의 PAUSED 오버레이는 **표시하지 않는다** (스킬 트리가 전체 오버레이를 대체). 스킬 트리 닫기 시 이전 상태(MARKET_OPEN)로 복귀.
- PRE_MARKET 상태에서는 `toggle_pause()` 호출 없이 스킬 트리만 열림 (시계가 이미 멈춰있으므로).

스킬 노드 상태 표현:
- `✓` (해금됨): 밝은 금색/흰색, 효과 활성
- `■` (해금 가능): 밝은 테두리 + 펄스 글로우 — SP 보유 + 선행조건 충족
- `□` (SP 부족): 보통 밝기 테두리 — 선행조건 충족, SP 미보유
- `○` (잠김): 어두운 회색 — 선행조건 미충족

스킬 선택/해금 플로우:
1. 노드 클릭 또는 방향키 이동 → 하단 상세 패널에 설명 표시
2. 해금 가능 상태면 `[해금하기]` 버튼 활성
3. 해금 확인 → 즉시 적용, `on_skill_unlocked` 시그널 발신
4. 해금 불가 시 사유 표시 ("선행 조건: A1 이동평균선 미해금")

#### 규칙 5. 시즌 종료 XP 정산 — 순차 공개

시즌 종료(`SEASON_END`) 결과 화면에 XP 정산을 순차 공개한다:

```
┌─────────────────────────────────────────────────────┐
│  시즌 3 종합 성적                                     │
│                                                      │
│  최종 순위: 3위 | 수익률: +25.3%                      │
│  총 자산: ₩1,253,000                                 │
│                                                      │
│  ───────────────────────────────────────────────────  │
│  시즌 XP 정산                    (0.5초 간격 순차 표시) │
│                                                      │
│  시즌 완주 보너스:         +200 XP                    │
│  순위 보너스 (3위):        +250 XP     ← 0.5s later  │
│  수익률 보너스 (+25%):     +250 XP     ← 0.5s later  │
│  ─────────────────────────────────────               │
│  총 시즌 XP:               +700 XP     ← 합산 등장   │
│                                                      │
│  ▐████████████████████████████████████████░░░░░▌     │
│  3,420 → 4,120 / 4,520 XP                           │
│  ───────────────────────────────────────────────────  │
│  [if 레벨업] ★ Lv 8 달성! 스킬 포인트 +1              │
│                                                      │
│              [ 다음 시즌 시작 ]  [ 스킬 트리 열기 ]    │
└─────────────────────────────────────────────────────┘
```

- 3줄 순차 공개 (0.5초 간격) → 기대감 조성
- 합산 등장 후 XP 바 채움 애니메이션
- 레벨업 시 배너 대신 인라인 표시 (시즌 결과 화면이 이미 풀스크린이므로)

#### 규칙 6. 미사용 스킬 포인트 알림

| 위치 | 표시 방식 |
|------|-----------|
| XP 바 (상시) | `[SP: N]` 배지, SP > 0일 때만 표시. 첫 등장 시 페이드인 |
| 장전 화면 (PRE_MARKET) | "미사용 스킬 포인트 N개 — [스킬 트리 열기 K]" 안내 문구 |
| 레벨업 배너 | `[스킬 트리 열기]` CTA 버튼 |

트레이딩 중 애니메이션/펄스 없음 — 전문적 톤 유지.

### States and Transitions

| UI 상태 | 트리거 | 표시 요소 |
|---------|--------|-----------|
| TRADING | 장 중 (F1 탭 활성) | XP 바 (정적) |
| GROWTH_TAB | F3 탭 클릭 (`MainScreen` 탭 전환) | GrowthScreen 전체 표시. 장 중 싱글플레이어이면 `MainScreen`이 자동 일시정지 발동 (`league-ui.md §3-1` 탭 전환 일시정지 정책 적용) |
| SETTLEMENT | `MARKET_CLOSED` | 정산 팝업 + XP 행 |
| LEVEL_UP | 레벨업 발생 + 정산 팝업 닫기 | 상단 배너 |
| SKILL_TREE | `K`키 / SP 배지 클릭 (현재 탭 위에 오버레이) | 풀스크린 오버레이 + 자동 정지 |
| SEASON_END | `SEASON_END` | 시즌 결과 + XP 순차 공개 |
| PRE_MARKET | 장전 대기 | SP 알림 문구 (SP > 0일 때) |

### Interactions with Other Systems

| 시스템 | 방향 | 인터페이스 |
|--------|------|-----------|
| XP 시스템 | → UI | `get_total_xp()`, `get_current_level()`, `get_xp_progress()`, `get_cumulative_xp_for_level()`, `get_daily_xp_breakdown() → { base_xp: int, multiplier: float, total_xp: int, return_tier: string }`, `on_xp_gained`, `on_level_up` |
| 스킬 트리 | ↔ UI | `get_all_skills()`, `get_skill_state()`, `unlock_skill()`, `get_available_skill_points()`, `on_skill_unlocked` |
| 게임 시계 | → UI | `on_market_state_changed(MARKET_CLOSED)` → 정산 트리거, `on_market_state_changed(SEASON_END)` → 시즌 정산, `toggle_pause()` → 스킬 트리 오버레이. 참고: game-clock.md는 편의 시그널 `on_market_close()`, `on_season_end()`도 제공하나, 프로그레션 UI는 `on_market_state_changed`를 단일 구독하여 상태 전환을 처리한다 (game-clock.md Signal Catalog 참조). |
| 트레이딩 스크린 | ↔ UI | XP 바는 trading_screen에 자식으로 추가, 정산 팝업에 XP 섹션 삽입 |
| 테마 시스템 | → UI | 색상/폰트 일관성 (ThemeSetup) |

## Formulas

> **Note:** 이 시스템은 순수 표시(Presentation) 레이어입니다. 모든 계산 값은 상위 시스템에서 사전 연산됩니다.

정산 팝업의 XP 공식 표시(기본 N × 배율 M = 결과)는 `XpSystem.get_daily_xp_breakdown()`에서 사전 계산된 값을 읽어 표시한다. UI가 직접 배율을 계산하지 않는다.

| 표시 데이터 | API | 반환 타입 |
|------------|-----|---------|
| XP 바 채움 비율 | `XpSystem.get_xp_progress()` | `float` (0.0~1.0) |
| 현재 레벨 | `XpSystem.get_current_level()` | `int` |
| 미사용 스킬 포인트 | `XpSystem.get_available_skill_points()` | `int` |
| 스킬 상태 | `SkillTree.get_skill_state(id)` | `LOCKED` / `UNLOCKED` + 추가 표시 상태 |

## Edge Cases

| 상황 | 처리 |
|------|------|
| 거래 0건인 날 장 마감 | 정산 팝업에서 XP 섹션 생략 또는 "거래 없음" 표시 |
| 한 번에 2+ 레벨업 | 배너: "Lv 5 → Lv 7 (+2)" 한 번만 표시, SP 합산. XP 시스템이 `on_xp_gained` 후 `on_level_up`을 순차 발신할 때, UI는 `_pending_level_ups: Array`에 수신된 레벨업을 누적한다. `on_xp_gained` 처리(정산 팝업 XP 바 애니메이션) 완료 후 `_pending_level_ups`를 확인하여 단일 배너로 합산 표시한다. |
| 시즌 보너스로 3+ 레벨업 | 시즌 결과 화면에 인라인 "Lv 5 → Lv 8 (+3)" |
| SP 0인데 스킬 트리 열기 | 정상 열림. 해금 버튼 비활성. "스킬 포인트가 필요합니다" 안내 |
| 모든 스킬 해금 상태 | SP 배지 표시 안 함. 스킬 트리에서 "모든 스킬 해금 완료" 메시지 |
| 정산 팝업 중 K키 | K키 무시. 정산 플로우 완료 후에만 스킬 트리 열기 가능 |
| XP 바 1080p 이하 | 바 높이 12px로 축소, 수치 텍스트 생략 (레벨 + 바만 표시) |

## Visual/Audio Requirements

- XP 바 색상: 금색 계열 (ThemeSetup.BTN_ACCENT 활용)
- 레벨업 배너: 금색 배경 + 흰색 텍스트
- 스킬 트리 배경: BG_DARK + 블러
- 해금 가능 노드: 펄스 글로우 (금색, 1.5초 주기)
- 해금 완료 노드: 밝은 금색/흰색
- 잠김 노드: 어두운 회색
- 효과음 (오디오 시스템 구현 시): 레벨업 알림음, 스킬 해금 확인음

## UI Requirements

- 키보드 조작: `K` → 스킬 트리 토글, `Esc` → 닫기, 방향키 → 노드 이동, `Enter` → 해금
- 마우스 조작: 노드 클릭 → 선택, 더블클릭 또는 해금 버튼 → 해금
- 접근성: 스케일러블 텍스트, 컬러블라인드 모드에서 상태 구분 (색상 + 아이콘 이중 표시)
- 최소 해상도: 1280×720 (XP 바 축소 모드)
