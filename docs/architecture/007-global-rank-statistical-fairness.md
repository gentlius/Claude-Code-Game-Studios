# ADR-007: 글로벌 순위 단순 return_pct 정렬 + AI 수익률 분포 단조성 보장

| Field | Value |
|-------|-------|

| **Date** | 2026-04-03 |
| **Decision Maker** | user + technical-director + game-designer |

## Context

20,000명 리그에서 플레이어의 글로벌 순위를 어떻게 산정할 것인가.

리그는 11개 티어(브론즈~거장)로 구성되며, 각 티어의 AI 참가자 수와
기대 수익률 범위가 다르다. 글로벌 순위는 티어 경계를 초월하여
플레이어가 20,000명 중 몇 위인지를 보여주는 수치다.

**핵심 긴장**: 고티어 AI는 저티어 AI보다 높은 수익률 분포를 가지도록 설계된다.
그런데 "정렬만 하면 공정한가?" — 저티어 AI가 운 좋게 고티어 AI보다 높은
return_pct를 기록할 경우 어떻게 처리할 것인가?

두 가지 접근이 충돌한다:
- **단순 정렬**: 누가 어느 티어든 return_pct 숫자만 비교
- **티어 보정 정렬**: 티어 내 백분위를 글로벌 점수로 환산하여 비교

## Decision

**전체 20,000명을 return_pct 단일 기준으로 정렬**한다.
티어 보정이나 백분위 환산 없이, 수익률 숫자 하나로 순위를 결정한다.

공정성은 정렬 방식이 아니라 **AI 파라미터 설계**로 보장한다:
티어가 높을수록 `mu`와 `r_min`이 단조 증가하도록 TIER_PARAMS를 설계하여,
확률적으로 고티어 AI가 글로벌 순위 상단을 점하는 구조를 만든다.

### 단조성 보장 파라미터 (TIER_PARAMS)

| 티어 | mu (%) | sigma (%) | r_min (%) | r_max (%) |
|------|--------|-----------|-----------|-----------|
| 브론즈 (0) | 100 | 60 | -30 | 600 |
| 실버 (1) | 130 | 55 | -25 | 650 |
| 골드 (2) | 160 | 45 | -20 | 550 |
| 플래티넘 (3) | 190 | 40 | -15 | 500 |
| 에메랄드 (4) | 215 | 35 | -10 | 450 |
| 다이아 (5) | 235 | 30 | -5 | 420 |
| 마스터 (6) | 250 | 25 | 0 | 380 |
| 그랜드마스터 (7) | 265 | 22 | +5 | 360 |
| 챌린저 (8) | 278 | 18 | +10 | 320 |
| 레전드 (9) | 290 | 15 | +15 | 300 |
| 거장 (10) | 310 | 20 | +50 | 500 |

`mu`와 `r_min` 모두 티어 순서대로 단조 증가한다.
`r_min` 단조 증가는 고티어 AI가 저수익 구간에 진입하는 것을 제한하며,
고티어 AI가 글로벌 순위 상단을 점하는 확률적 보장을 제공한다.

### 글로벌 순위 계산 방식

```
global_rank = (모든 티어 AI + 플레이어를 return_pct 내림차순 정렬).index(플레이어) + 1
```

전체 정렬은 시즌 종료 시 1회 수행한다 (정확한 최종 순위 결정).
일 중 글로벌 순위 추정은 ADR-004에서 정의한 버킷 기반 추정(O(log N))을 사용한다.

### AC-02 단조성 테스트

`AiCompetitor._validate_tier_monotonicity()`가 `init_season()` 시점에
런타임으로 검증하며, 위반 시 `push_error`로 즉시 경고한다.

```gdscript
for t in range(TIER_COUNT - 1):
    assert TIER_PARAMS[t + 1]["r_min"] >= TIER_PARAMS[t]["r_min"]
    assert TIER_PARAMS[t + 1]["mu"]    >= TIER_PARAMS[t]["mu"]
```

## Alternatives Considered

### A. 티어 내 백분위 기반 글로벌 환산

- **설명**: 각 AI/플레이어의 티어 내 백분위를 계산하고, 이를 글로벌 점수로
  환산하여 정렬. 예: 브론즈 90%ile = 글로벌 90점, 다이아 50%ile = 글로벌 75점
  (가중치 공식 필요).
- **장점**: 티어 간 분포 차이를 보정하여 "같은 노력"이 유사한 순위를 보장.
- **단점**: 글로벌 점수 환산 공식이 복잡. 플레이어가 이해하기 어려움
  ("내 수익률이 +150%인데 왜 브론즈 유저보다 글로벌 순위가 낮지?").
  공식 파라미터(가중치)가 추가 밸런스 튜닝 대상이 됨.
  직관적이지 않은 순위가 플레이어에게 혼란 또는 불공정감을 줄 수 있음.
- **기각 이유**: 복잡성 + 직관성 부족. 공정성은 파라미터 설계로 달성 가능.

### B. 티어별 정규화 점수 + 가중 합산

- **설명**: (티어 가중치 × 티어 내 백분위)로 글로벌 점수 산정.
  고티어일수록 가중치가 높아 상위 글로벌 순위 점유 유도.
- **장점**: 티어 구조를 순위에 명시적으로 반영.
- **단점**: "왜 같은 수익률인데 티어에 따라 글로벌 순위가 달라지나?"라는
  플레이어 불만 가능성. 가중치 파라미터 추가 튜닝 부담.
  return_pct가 동일한 두 플레이어의 순위가 티어에 따라 다른 것은
  일부 플레이어에게 불공정으로 느껴질 수 있음.
- **기각 이유**: 정렬 방식의 복잡성이 파라미터 설계 단조성 보장보다 비용이 큼.

## Consequences

### 긍정적

- 순위 산정 방식이 "수익률 높은 사람이 위"로 플레이어에게 직관적
- 글로벌 순위 계산 코드가 단순 (정렬 1회)
- 공정성 보장 책임이 TIER_PARAMS 설계에 집중되어 밸런스 조정 포인트가 명확
- `_validate_tier_monotonicity()` 런타임 검증으로 파라미터 설계 오류를 즉시 감지

### 부정적

- 저티어 AI가 고분산(sigma) 구간에서 운 좋게 고수익을 기록하면
  고티어 AI보다 글로벌 순위가 높을 수 있음 (확률적 이상값)
  → r_min 단조 증가로 출현 빈도는 낮추지만 완전히 차단하지는 않음
- 브론즈 AI 수가 7,600명으로 가장 많아 글로벌 하위권 대부분을 브론즈가 채움
  (의도된 설계 — 하위권은 저티어가 점유하는 것이 자연스러움)

### 리스크

- **mu 교대 패턴 위반**: 복리 공식 기반 mu 계산 시 티어 배율(3x vs 3.33x) 교대로
  인해 mu가 단조 증가하지 않는 현상이 발생했음 (Sprint 2 중 발견).
  완화: 선형 보간 방식으로 mu를 재산정하여 단조성 확보. 현재 TIER_PARAMS가
  해당 수정 이후의 값이며, `_validate_tier_monotonicity()`가 회귀를 방지.
- **거장 티어(10) r_max 상향**: 거장 AI r_max=500%로 다른 고티어보다 높게 설정됨.
  글로벌 1위가 거장 AI로 고정될 가능성이 높음. 이는 의도된 설계
  ("거장은 넘을 수 없는 존재")이나, 플레이어가 1위에 도달하는 에지 케이스 주의.

## Performance Implications

- **CPU**: 시즌 종료 시 20,000개 float 정렬 = O(N log N) ≈ 20,000 × 15 = 300,000 연산.
  시즌 종료는 이벤트성(1회)이므로 프레임 예산 영향 없음.
- **Memory**: 정렬 임시 배열 20,000 × 8바이트 ≈ 160KB. 무시 가능.
- **Load Time**: 영향 없음.
- **Network**: 해당 없음.

## Validation Criteria

- **AC-01**: 모든 티어에서 `TIER_PARAMS[t+1]["r_min"] >= TIER_PARAMS[t]["r_min"]`
  및 `TIER_PARAMS[t+1]["mu"] >= TIER_PARAMS[t]["mu"]` 성립.
  `_validate_tier_monotonicity()` 통과.
- **AC-02**: 동일 시즌 시드에서 100회 시뮬레이션 후, 글로벌 순위 상위 1%의
  티어 분포에서 다이아 이상이 80% 이상을 차지함 (밸런스 검증 테스트).
- **AC-03**: 시즌 종료 시 `global_rank` 계산 결과가 `[1, 20000]` 범위 내.
- **AC-04**: 플레이어 return_pct가 모든 AI보다 높을 때 `global_rank = 1`.
