> **Status**: Active

---

## 목적

Beta 판정 기준인 **"3시즌 플레이 후 4시즌을 시작하고 싶은가"** 를 객관적으로 측정한다.
튜토리얼 없이 처음 해보는 플레이어가 3시즌 이상 자연스럽게 플레이할 수 있는지 확인한다.

---

## 대상 및 인원

| 항목 | 기준 |
|------|------|
| 인원 | 최소 3명 (합격 판정: 3명 중 2명 이상) |
| 대상 | 게임 개발 경험 없는 성인 (실제 잠재 플레이어군) |
| 사전 준비 | 없음 — 인게임 설명만 허용 |

---

## 세션 설계

### 1단계: 자유 플레이 (60~90분)

- 진행자는 관찰만 수행. 질문에 "스스로 알아보세요"로 응답.
- 3시즌 완주 또는 60분 중 먼저 도달하는 시점에 종료.
- 관찰 항목:
  - 막히는 UI 지점 (스크린샷 메모)
  - 오해 또는 잘못 이해한 규칙
  - 레벨업 시 반응
  - 시즌 종료 시 반응 ("다음 시즌 하고 싶다" 발화 여부)

### 2단계: 설문 (5분)

플레이어에게 0~5점 척도로 응답받는다.

| 질문 | 척도 |
|------|------|
| Q1. "다음 4시즌을 지금 바로 시작하고 싶다" | 0 (전혀 아니다) ~ 5 (매우 그렇다) |
| Q2. "게임 규칙이 명확하게 이해됐다" | 0 ~ 5 |
| Q3. "자산이 성장하는 느낌이 있었다" | 0 ~ 5 |
| Q4. "다른 사람에게 추천하고 싶다" | 0 ~ 5 |

## 이중 승리 조건 검증 시나리오 (QA Lead 직접 수행)

플레이테스터 세션과 별개로 QA Lead가 다음 두 경로를 수동 시나리오로 검증한다.

### 루트 A — 현금 자산 ≥ 100,000,000,000원 (1,000억)

1. 새 게임 시작 → 플레이어 세션 진행 (`active_slot_id` 확인)
2. 시즌 반복으로 `cash_assets ≥ 100,000,000,000` 달성
3. 게임 내 "거장" 엔딩 트리거 확인
4. 엔딩 화면에서 "루트 A" 텍스트 또는 명확한 승리 표시 확인

### 루트 B — 총 자산 ≥ 1,000,000,000,000원 (1조)

1. 새 게임 시작 (루트 A와 별도 슬롯)
2. `total_assets ≥ 1,000,000,000,000` 달성
3. "거장" 엔딩 트리거 확인
4. 엔딩 화면 확인

> 위 시나리오는 실제 플레이 달성이 어려울 수 있으므로 SaveSystem 또는 테스트 모드로
> 자산을 직접 설정 후 검증해도 무방. 조건 판정 로직의 정확성을 검증하는 것이 목적.

## 가격 정찰 익스플로잇 검증 (ADR-018)

1. 특정 Day N에서 종목 가격 시퀀스를 기록
2. 동일 세이브 파일 5회 로드
3. 각 로드 후 Day N 가격 시퀀스가 모두 다름을 확인
4. 5회 중 동일한 시퀀스가 2회 이상 → 실패 (ADR-018 구현 불완전)

## A3 재무제표 검증 (QA Lead 직접 수행)

| 단계 | 행동 | 합격 기준 |
|------|------|----------|
| 1 | A3 미해금 상태에서 임의 종목 선택 → 종목 패널 확인 | 재무 지표 섹션 미표시 |
| 2 | A3 해금 → 동일 종목 패널 재확인 | PER·PBR·ROE·배당 4개 값 표시 |
| 3 | 결손 기업(바이오 적자사 등) 종목 선택 | PER "N/A" 표시 |
| 4 | 무배당 종목 선택 | 배당 행 미표시 |
| 5 | 10틱 관찰: PER·PBR·ROE 값 변화 여부 | 가격 변화 시 PER·PBR 비례 증가, ROE 불변 |
| 6 | `--export-release` 빌드 후 5초 생존 + 로그 확인 | SCRIPT ERROR 없음 |

## S3 루머 채널 검증 (QA Lead 직접 수행)

| 단계 | 행동 | 합격 기준 |
|------|------|----------|
| 1 | S2까지만 해금 상태로 10분 플레이 | 루머 카드(회색 배경 + `[루머]` 태그) 미표시 |
| 2 | S3 해금 → 뉴스 발동 틱 기록 → (발동 틱 - 60) 시점 확인 | 해당 틱에 루머 카드 표시 |
| 3 | 루머 카드 시각 확인 | 일반 뉴스 카드와 배경색·태그로 구분됨 |
| 4 | 장 마감 60틱 이내 루머 발화 관찰 | 다음 거래일에 해당 뉴스 미발동 (취소 처리 확인) |

> 정확도 70% 수렴(AC-04)은 유닛 테스트로 대체 — `test_rumor_accuracy()` 통과 여부 확인.

## 라이프스타일 비시즌 정산 검증 (QA Lead 직접 수행)

| 단계 | 행동 | 합격 기준 |
|------|------|----------|
| 1 | 부동산 1개 보유 상태로 시즌 종료 | 비시즌 윈도우에서 임대 수익이 cash_assets에 반영됨 |
| 2 | 라이프스타일 화면에서 항목 구매 | 구매 즉시 잔여 자산 감소, "소비 후 잔여" 실시간 갱신 |
| 3 | 거주지 업그레이드 구매 | "이사 날" 페이드 블랙 → 새 배경 페이드인 → 타이틀 카드 재생 |
| 4 | F3 화면 진입 | 현재 거주지 이미지로 배경 표시 |
| 5 | 구매 확정 → 앱 재시작 | 소비 내역·거주지 유지 (세이브/로드 정합성) |
| 6 | cash_assets < ₩1,000,000 상태로 "다음 시즌" 진입 시도 | 프리마켓 진입 경고 표시, 경고 후 진행은 가능 |
| 7 | Recurring 비용 납부 불가 상태로 시즌 종료 | 경고 + 해당 항목 강제 해지, 환불 없음 확인 |
| 8 | "다음 시즌 시작" 즉시 클릭 (소비 없이 스킵) | 임대 수익 등 비시즌 정산은 자동 처리됨 |
